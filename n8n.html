n8n

پلتفرم اتوماسیون کاری مبتنی بر نود - Fair-Code Workflow Automation

  1. خانه
  2. n8n

n8n (بخوانید "n-eight-n") یک ابزار قدرتمند و منعطف اتوماسیون workflow است که به کاربران امکان می‌دهد تا فرآیندهای کاری خود را با اتصال سرویس‌های مختلف به یکدیگر اتوماتیک کنند. این پلتفرم با رابط کاربری visual و node-based، امکان ساخت workflow های پیچیده را بدون نیاز به کدنویسی فراهم می‌کند.

ویژگی‌های کلیدی

🔗 300+ Integration

اتصال با بیش از 300 سرویس مختلف شامل Google Workspace، Slack، GitHub، و غیره

🎨 Visual Workflow Builder

ساخت workflow های پیچیده با رابط drag & drop بدون نیاز به کدنویسی

🔧 Custom Code Support

امکان اجرای کد JavaScript، Python و Function nodes برای منطق پیشرفته

📊 Data Processing

تبدیل، فیلتر و پردازش داده‌ها بین سرویس‌های مختلف

⏰ Scheduling & Triggers

اجرای workflows بر اساس زمان‌بندی، webhook ها یا تغییرات داده

🏢 Self-Hosted

کنترل کامل بر داده‌ها با امکان میزبانی در سرور شخصی

نصب و راه‌اندازی

# نصب سریع با Docker docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n # یا با volume برای ذخیره داده‌ها docker run -it --rm --name n8n -p 5678:5678 -v ~/.n8n:/home/node/.n8n n8nio/n8n

⚠️ نکات امنیتی

  • احراز هویت: حتماً Basic Auth یا OAuth را فعال کنید
  • HTTPS: در محیط production از SSL استفاده کنید
  • Environment Variables: اطلاعات حساس را در متغیرهای محیطی نگه دارید
  • Network Security: دسترسی به پورت n8n را محدود کنید
  • Database Security: رمز عبور قوی برای دیتابیس تنظیم کنید
  • Webhook Security: از webhook authentication استفاده کنید
  • Backup: بکاپ منظم از workflows و دیتابیس تهیه کنید

سوالات متداول

n8n متن‌باز است و می‌توانید آن را در سرور خود نصب کنید، در حالی که Zapier یک سرویس ابری است. n8n قابلیت سفارشی‌سازی بیشتری دارد و امکان اجرای کد JavaScript را فراهم می‌کند.

بله، رابط visual n8n برای مبتدیان مناسب است. اما برای workflows پیچیده ممکن است نیاز به یادگیری مفاهیم پایه داشته باشید. مستندات و template های آماده کمک زیادی می‌کند.

بله، با نصب self-hosted کنترل کامل بر داده‌ها دارید. فقط باید نکات امنیتی مثل HTTPS، احراز هویت قوی و رمزنگاری را رعایت کنید.

می‌توانید workflows را به صورت JSON export کنید یا از volume دیرکتوری ~/.n8n بک‌آپ تهیه کنید. همچنین می‌توانید از Git برای version control استفاده کنید.
